变量

​ 变量是程序中存储数据的基本单元，编程的本质：对变量进行赋值、判断、处理。程序可以总结为：

​ 程序 = 变量 + 算法（处理逻辑）

所以本节介绍Go的变量，目的学习：

  1. Go 的变量声明及命名规范。
  2. Go 的常量变量。
  3. Go 的变量作用域。
  4. Go 的变量内存类型。

1.2.1 变量声明​

1.2.1.1 变量​

变量的本质就是进程中的一小块内存，用于存储数据。变量要素包含：

  1. 变量名称
  2. 类型
  3. 值

1.2.1.2 声明​

变量声明的标准格式：

  1. 未初始化： var 变量名 变量类型
  2. 初始化：var 变量名 变量类型 = 表达式 或者 变量名:=表达式
  3. 类型推断： var 变量名 = 表达式
// 声明，未初始化
var age int
// 变量赋值
age = 10
// 声明，并初始化
var money int = 10
// 同类型
var name,addr string
// 匿名声明
sum := 10
// 类型推断，i 的类型与sum 一样
var i = sum

1.2.1.3 默认值​

针对未初始化的变量，其默认值：

  1. 基本类型中的整型、浮点类型变量默认值:0；
  2. 布尔类型变量默认值:false;
  3. 函数、指针变量、切片 默认值: nil

【推荐】：变量声明，并且初始化（C/C++ 编程经验迁移过来）。

1.2.1.4 匿名变量​

匿名变量即不占用命名空间，也不会分配内存。匿名变量符号:“_”, 用于函数多值返回，被舍弃的值。

package main
func echo(world string)(string, int){
  newWorld := fmt.Sprintf("Hello %s", world)
  return newWorld,1
}
func main(){
  // echo 的另一个int类型返回值，不需要，所以通过匿名变量“_”舍弃。
  v,_ := echo("Go")
  fmt.Println(v)
}

1.2.1.5 命名规范​

  1. 变量命名 由字母、数字、下划线_组成。

  2. 变量首字母必须是非数字。

Go 语法规定，定义的局部变量如果没有被使用会发生编译错误。

func say(){
   // 声明了变量，但没有被使用，编译器会报错，集成环境会显示红色。
   var age int
}

1.2.2 变量作用域​

​ 作用域为已声明标识符所表示的常量、类型、变量、函数或包在源代码中的作用范围和生命周期。一般来说有三个地方可以定义变量：

  • 在函数或一个代码块内部声明的变量，称为局部变量。
  • 在函数参数的定义中声明的变量，称为形式参数。
  • 在所有函数外部声明的变量，称为全局变量。

作用域是程序的一个区域，变量的作用域可以分为以下几种：

  • 局部作用域：在函数内部声明的变量具有局部作用域，它们只能在函数内部访问。局部变量在函数每次被调用时被创建，在函数执行完后被销毁。
  • 全局作用域：在所有函数和代码块之外声明的变量具有全局作用域，它们可以被程序中的任何函数访问。全局变量在程序开始时被创建，在程序结束时被销毁。
  • 块作用域：在代码块内部声明的变量具有块作用域，它们只能在代码块内部访问。块作用域变量在代码块每次被执行时被创建，在代码块执行完后被销毁。
  • 类作用域：在类内部声明的变量具有类作用域，它们可以被类的所有成员函数访问。类作用域变量的生命周期与类的生命周期相同。

根据变量的作用域，变量分为：

  1. 局部变量，局部变量的域在某个{}内。

  2. 全局变量，全局变量的域是全局的（在某个go代码文件中）。

  3. 形势参数，在某个函数内生效。

  4. 例子

    package main

    import "fmt"
    // 声明并初始化全局变量
    var gname string = "Ruo"

    /**
    * 全局变量不能通过简单方式进行声明，不然会报错
    *g2 := "rss"
    */

    func say(world string){
        // say 内的局部变量
        var age int = 10
      // age生效的值是10
        fmt.Printf("1>> Hello %s age:%d\n", world, age)
        if true{
            // 更小作用域的变量
            var age int = 11
        // age 生效的是值11 的变量
            fmt.Printf("2>> Hello %s age:%d\n", world, age)
        }
      // age生效的值是10
        fmt.Printf("3>> Hello %s age:%d\n", world, age)
    }

    func main() {
        var age int
        var money int = 11
        age = 10
      fmt.Printf("gname=%s,age=%d, money=%d\n", gname, age, money)
    }

​ 以上输出：

​ gname=Ruo,age=10, money=11 ​ 1>> Hello Go age:10 ​ 2>> Hello Go age:11 ​ 3 >> Hello Go age:10

1.2.3 常量变量​

​ 通过const 修饰的变量声明都是常量变量，常量变量声明之后不能被修改，因此常量变量声明要用初始化声明。

​ 其声明标准方式：

  1. 声明单个常量变量: const 变量名 变量类型 = 表达式

  2. 声明多个常量变量: const (变量名 变量类型 = 表达式 换行 变量名 变量类型 = 表达式)

  3. 参考例子：

    // 声明常量变量: 
    const (
        ST_OK        int = 0
        ST_FAILED    int = -1
        ST_NOT_SOUND int = 400
    )
    // 声明单个
    const DF_NAME = "Ruo"

1.2.4 变量内存​

变量的存储内存，在计算机中有两种：

  1. 栈内存（线性连续的内存）。

    栈内存由编译器自动分配和释放，开发者无法控制。栈内存一般存储函数中的局部变量、参数等，函数创建的时候，这些内存会被自动创建；函数返回的时候，这些内存会被自动释放。

  2. 堆内存。

    在C/C++ 中，通过malloc 或者new 创建的内存就是堆内存（由于C/C++没有垃圾回收，所以得自己去释放）；在Go 中通过new 出来的内存。堆内存的生命周期比栈内存要长，堆适合不可预知大小的内存分配。在Go 中，如果函数返回的值还会在其他地方使用，那么这个值就会被编译器自动分配到堆上（内存逃逸）。堆内存相比栈内存来说，不能自动被编译器释放，只能通过垃圾回收器才能释放，所以栈内存效率会很高。

在Go中：

  1. 【栈内存】的分配和释放都是对虚拟内存的操作。
  2. 【堆内存】的分配和释放都是对虚拟内存的操作。
