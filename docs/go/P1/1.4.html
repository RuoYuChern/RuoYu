指针与函数

本章的学习目的

  1. 了解指针
  2. 了解函数，及其参数传递。
  3. 了解内存逃逸

1.4.1 指针​

1.4.1.1 声明​

变量是一种使用方便的占位符，其实质指向计算机的内存地址。指针变量也是变量，只不过其直接指向的内存地址保存着另外一个变量的内存地址。

  1. Go 指针只能取值运算： *变量，获取该指针值指向的内存对应的变量值。

  2. 不能对Go指针进行运算，比如p++/p--，p是指针变量。

  3. 未初始化的指针，其值是nil，这也是未初始化的切片(slice)、map 变量其值是nil。未初始化的指针不能操作。

  4. 指针声明: var ptr *Type

  5. 对变量进行取指采用&。

  6. 如图是指针变量和非指针变量的区别，即指针变量A其访问到的值是另外变量B的内存地址，如果要通过指针变量A访问到B的值，择要采用取指运算*A。变量C、变量B都是直接访问到值。

    func printPointer() {
        // 声明一个int32的指针，其默认值为nil
        var iPtr *int32
        // 判断指针是否空
        if iPtr == nil {
            fmt.Printf("iPtr is nil:%p\n", iPtr)
        }
        // 声明并初始化一个int32变量
        var i int32 = 10
      //注不用这个来声明int32变量，这个声明的是int变量： i := 10
      // 通过取指&运算，将i的地址赋值给iPtr
        iPtr = &i
      // 打印变量iPtr的值及其指向的值
        fmt.Printf("iPtr value:%v, pointer to value:%d\n", iPtr, *iPtr)
    }
    /*
    Ptr is nil:0x0
    iPtr value:0xc000018178, pointer to value:10
    */

1.4.1.2 指针数组​

指针数组：元素为指针的数组，其声明方式：

var  ptr [N]*Type
/**声明并初始化数组**/
var iar = [4]int32{5,6,7,8}
/**声明数组指针**/
var ptr [4]*int32

// 赋值
for i:=0; i < len(iar);i++{
   ptr[i] = &iar[i]
}

// 使用
for i:=0; i < len(ptr);i++{
  fmt.Ptrinf("iar[%d] = %d", i, *ptr[i])
}

1.4.1.3 指针的指针​

指针的指针，其实双重指向。即一个指针变量的值存储着另外一个指针变量的地址。

  1. ptrA 得到的是ptrB变量地址。
  2. *ptrA 得到的是ptrB 指向的地址，即C的地址。
  3. **ptrA 得到的是C的值。
var c int32 = 100
// ptrB 指向
var ptrB *int32 = &c
// ptrA 指向ptrB
var ptrA **int32 = &ptrB 
fmt.Printf("&ptrB: %p, %p\n", &ptrB, ptrA)
fmt.Printf("&c: %p, %p\n", &c, *ptrA)
fmt.Printf("c: %d, %d\n", c, **ptrA)
/*
&ptrB: 0xc000012030, 0xc000012030
&c: 0xc00001817c, 0xc00001817c
c: 100, 100
*/

1.4.1.4 指针转换​

​ 类似C/C++ 语言中的long，Go中的uintptr 整型可以用于存放任何指针。在Go中，我们不能进行指针运算，但是借助于uintptr就可以进行指针运算。但普通类型指针不能直接转换成uintptr，必须先转换成unsafe.Pointer类型之后，再转换成uintptr。

uintptr

// uintptr is an integer type that is large enough to hold the bit pattern of
// any pointer.
// 计算机内存地址的长度，取决于CPU的位数，32的CPU是4字节，64位的是8字节。
type uintptr uintptr

unsafe.Pointer

// 定义了非类型安全指针
type ArbitraryType int
type Pointer *ArbitraryType
// 提供了操作函数
func Alignof(variable ArbitraryType) uintptr // 用来获取变量variable的对齐保证
func Offsetof(selector ArbitraryType) uintptr // 用来获取结构体值中的某个字段的地址相对于此结构体值地址的偏移
func Sizeof(variable ArbitraryType) uintptr // 用来获取变量variable变量的大小尺寸

通过unsafe.Pointer 普通类型指针 与uintptr 之间的转换：

使用注意事项

  1. 通过uintptr转换回unsafe.Pointer时，其不能赋值给一个变量进行中转，不然可能访问到无效的指针。因为在访问该指针的时候，其指向的变量可能被Go回收。

参考：https://golang.google.cn/pkg/unsafe/#Pointer，给出6种安全的使用方式：

// 通过非类型转换 将T1 转换成T2
// 该例子通过指针转换：将float64 转换uint64
func Float64bits(f float64) uint64 {
    return *(*uint64)(unsafe.Pointer(&f))
}

// 指针运算：p = unsafe.Pointer(uintptr(p) + offset)
// 通过指针运算，取得对象s的成员f 指针值。
f := unsafe.Pointer(uintptr(unsafe.Pointer(&s)) + unsafe.Offsetof(s.f))
// 通过指针运算，取得数组x的第i个元素x[i]指针值
e := unsafe.Pointer(uintptr(unsafe.Pointer(&x[0])) + i*unsafe.Sizeof(x[0]))
//通过指针移动到变量内存地址的末尾是无效的：
// INVALID: end points outside allocated space.
var s thing
end = unsafe.Pointer(uintptr(unsafe.Pointer(&s)) + unsafe.Sizeof(s))
// INVALID: end points outside allocated space.
b := make([]byte, n)
end = unsafe.Pointer(uintptr(unsafe.Pointer(&b[0])) + uintptr(n))

1.4.2 函数​

1.4.2.1 声明​

函数是指完成一定功能的代码块，具有如下的元素：

  1. 函数名称
  2. 函数入参
  3. 函数出参： 通过return 返回
  4. 函数体（实现)
/**
* 函数名称:funtionName
* 参数： s1, s2,....
*/
func funtionName(s1 T1, s2 T2,....)(T1,...){
  return ...
}
/**
* 声明一个没有参数/出参的函数
*/
func funtionName(){
  
}
/**
* 声明一个入参类型一样的函数
*/
func funtionName(s1,s2 T){
  
}

func sum(a1 int32, a2 int32)int32{
  return (a1 + a2) 
}

1.4.2.2 函数变量​

类似C/C++, Go中函数也是一种数据类型，可以声明变量。函数变量存储的是函数的入口地址。通过type 可以声明一个函数类型。

函数变量的使用场景：

  1. 回调，用于网络编程或者多线程。
  2. 泛型，根据不同的类型进行处理。
  3. 流水线处理(pipline)
type CallBack func(int32)

type CallBack func(int32)

func cb1(r int32) {
    fmt.Printf("cb1: %d\n", r)
}
func cb2(r int32) {
    fmt.Printf("cb2: %d\n", r)
}
func cb3(r int32) {
    fmt.Printf("cb3: %d\n", r)
}
func printFun() {
    mp := make(map[int32]CallBack)
    mp[10] = cb1
    mp[11] = cb2
    mp[12] = cb3
    for k, v := range mp {
        v(k)
    }
}
/**
cb3: 12
cb1: 10
cb2: 11
*/
type Filter func(int32)bool

func isBiggerThan100(r int32)bool{
  return (r >= 100)
}

func isOdd(r int32)bool{
  return ((r % 2) != 0)
}

// 处理链，指责链
var pl = [2]Filter{isOdd, isBiggerThan100}
var b int32 = 101
for i:=0; i < len(pl);i++{
  if(!pl[i](b)){
     break
  }
}

1.4.2.3 参数传递​

在Go中，函数参数（入参、出参）传递都是值传递。在参数传递都会发生变量声明及初始化。

  1. 基本类型参数，则进行简单赋值（浅拷贝）。
  2. 派生类型参数（对象，数组，字符串）则会进行深度赋值（深度拷贝）。
  3. 指针类型参数，则进行地址值赋值（地址值其实也是一个数字）（浅拷贝）。
  4. 对于数组、字符串、类作为函数参数，从性能角度来看建议采用指针。对于浅拷贝的传参就不建议指针传参。
// 值传递
func f1(r int){
   r = r + 10
}
var r int = 0
// 函数传参，是值传递，函数类型的修改 不会改变r的值。
f1(r)
// 值传递：r = 0

func f2(r *int){
   *r = *r + 10
}
// 函数传参，是值传递，但传递的是地址。
// 函数对指针所指向的内容进行修改，会影响到r
f2(&r)
// r = 20
func f3(r **int){
    *r = new(int)
    **r = 20
}
// 声明一个nil 指针变量
var pptr *int
f3(&pptr)
//pptr 指向一个指针，*pptr = 20
func f4()*int{
  r := new(int)
  *r = 40
  return r
}

1.4.2.4 内存逃逸​

对于C/C++ 如果将一个函数的栈对象变量作为指针返回，那么函数外访问该指针，会访问到野指针的方向，导致程序异常。

那么在Go呢？Go编译器针对这种情况，会将变量逃逸到堆上，由垃圾回收进行回收。

type Person struct{
   age int
   name string
}

func fp1()*Person{
  // 发生内存逃逸
  p :=&Person{age: 10, name:"RuoY",}
  return p
}

func fp2()*Person{
  // 发生内存逃逸
  p := new(Person)
  p.age = 20
  p.name = "RY"
  return p
}
// 不会发生内存逃逸
person := fp1()
//内存逃逸
fmt.Printf("%+v\n", *person)
// 不会发生内存逃逸
person = fp2()
//内存逃逸
fmt.Printf("%+v\n", *person)

通过go build -gcflags=-m xxx.go 进行编译，会输出内存逃逸信息。

./example.go:264:7: &Person{...} escapes to heap ./example.go:269:10: new(Person) escapes to heap

./example.go:302:15: &Person{...} does not escape ./example.go:303:12: ... argument does not escape ./example.go:303:22: *person escapes to heap

./example.go:304:14: new(Person) does not escape ./example.go:305:12: ... argument does not escape ./example.go:305:22: *person escapes to heap

1.4.2.5 内存逃逸的场景​

  1. 函数返回指针（指针逃逸），在其它地方继续使用。

  2. interface{} 动态类型逃逸，由于在编译时不知道变量的具体类型，也会发生逃逸。

  3. 栈空间不足。

  4. 函数闭包。

    func sum() func(n int) int {
      // 局部变量s 会发生内存逃逸
        s := 0
        return func(n int) int {
            s = n * (n + 1) / 2
            return s
        }
    }

    ss := sum()
    ss(10)
    ss(20)
